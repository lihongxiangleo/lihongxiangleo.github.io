Gradle基础及常用命令

Posted on 2016-10-13   |   In Gradle   |  

什么是Gradle

现在大部分的Android开发者所用的 IDE 应该都是 AndroidStudio 了，我们知道 AS 中的项目是基于 Gradle 构建的。Gradle 是一种基于 Groovy 语言的构建工具，Groovy 语法跟 Java 语法很相像，是一种动态语言，而且在 Java 基础上做了不少改进，用起来更加简洁、灵活。

Gradle Wrapper

现在默认新建一个项目，然后点击 AS 上运行，默认就会直接帮你安装 Gradle ，我们不需要额外的安装 Gradle 了，但是其实这个 Gradle 不是真正的 Gradle ，他叫 Gradle Wrapper ，意为 Gradle 的包装，什么意思呢？假设我们本地有多个项目，一个是比较老的项目，还用着 Gradle 1.0 的版本，一个是比较新的项目用了 Gradle 2.0 的版本，但是你两个项目肯定都想要同时运行的，如果你只装了 Gradle 1.0 的话那肯定不行，所以为了解决这个问题，Google 推出了 Gradle Wrapper 的概念，就是他在你每个项目都配置了一个指定版本的 Gradle ，你可以理解为每个 Android 项目本地都有一个小型的 Gradle ，通过这个每个项目你可以支持用不同的 Gradle 版本来构建项目。

Gradle常用命令

  • 查看构建版本 ./gradlew -v
  • 清除build文件夹 ./gradlew clean
  • 检查依赖并编译打包 ./gradlew build (会把debug和release包都打出来)
  • 编译并打Debug包 ./gradlew assembleDebug
  • 编译并打release包 ./gradlew assembleRelease
  • Release模式打包并安装 ./gradlew installRelease
  • 卸载Release模式包 ./gradlew uninstallRelease

assemble还可以结合productFlavor实现多渠道打包

注：./ 代表当前目录，gradlew 意为 Gradle Wrapper

选择Hexo搭建个人博客

来沪工作后所接手的项目一直在叠加需求，工作比较忙碌再加之自己的惰性，很久都没写博客来总结一下开发中遇到的坑和知识，之前曾在csdn上写博客：博客地址。但是由于csdn糟糕的体验和操蛋的服务器让我完全不能忍受，所以决定来搭建自己的个人博客了。如题我选择了Hexo这个博客框架，Hexo是一个快速、简洁且高效的博客框架，只需一条指令即可部署到 GitHubPages，Heroku 或其他网站，具体如何搭建和更换主题可以参照Hexo的官方文档，这里我选择的是简洁优雅的NexT主题。

  • Hexo官方文档
  • Next主题配置使用文
    在这里，要感谢Hexo和Next的作者！

Hexo常用命令

Hexo的命令非常的简洁，具体的命令在官方文档上都有,大家可以自行去学习：Hexo命令

常用的命令大概就下面几条：

  • hexo new title生成一篇名为“title”的新文章，采用默认的模板
  • hexo g 重新渲染Blog
  • hexo s –debug启动本地服务器，可以用来本地测试
  • hexo d 将本地public目录下的文件内容提交到Github，具体的项目地址在_config.yml设置
  • hexo clean 清空本地的缓存文件db.json和已生成的静态文件public

在本地测试的时候有可能会遇到端口被占用的问题，我目前的处理方式是关掉网页然后重启Terminal就解决了。一般的写作流程差不多是着这样的：
hexo new出一篇新的文章，写作完成后使用hexo g渲染，本地测试通过后使用hexo d部署到github page，完成文章的发布。

结尾

倒腾了一天半，终于把博客的基本功能搭建好了，可以正常发表文章，目前还没有集成评论功能和统计功能，之后再慢慢给弄上去。希望自己能够克服自己的惰性坚持写博客（毕竟我花了这么长的时间来搭建！！！），多积累多学习，我们开始吧骚年！
